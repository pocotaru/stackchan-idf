動作モード

Stack-chan の主な振る舞いを 3 つから選びます。「会話応答」は OpenAI / Gemini / XiaoZhi のいずれかで会話、「JTTS ランダム発話」は音声のアイドル発話のみ、 「マイクでリップシンク」はマイク入力に合わせて口だけ動かします。 保存後の再起動で切替わります。

空欄のままにすると現在の値を変更しません

デバイス名 (BLE / mDNS)

BLE アドバタイズ名と mDNS ホスト名のもとになります。空欄なら自動生成 (Stackchan-XXXXXX)。 mDNS は英数字とハイフンに丸められます。保存後の再起動で反映。

パスワード認証

設定すると、この設定ページ (WiFi) と BLE の両方で認証が必要になります。 空欄を保存すると認証なし (現状互換) に戻ります。HTTP は Basic Auth、BLE は 暗号鍵に混入されます。保存後の再起動で反映。

顔描画 DSL (Avatar)

アバターの顔の描画ロジックを Lua 風 DSL で書いて、ブラウザ内でバイトコード化 → 実機に送信して即時反映します。NVS に保存されるので再起動後も維持。 文法ガイド ／ デフォルト顔ソース

DSL コンパイラ未起動

スピーカー音量

内蔵 / Module Audio / Atomic Echo Base すべての出力に効きます。 スライダーを動かすと 即時反映 + 自動保存 (再起動不要)。 100% = ボード ごとの工場既定値、200% でブースト。

jtts 試し発話

ひらがな / カタカナで入力して「話す」ボタンを押すと jtts で読み上げます。 漢字や英字は変換されないので、フリガナを直接入力してください。 音量や音声出力の確認に便利。

音声出力

音声の出力先を選びます。「自動」は M5 Module Audio が装着されていれば そちらを優先、未装着なら内蔵スピーカ。「内蔵」と「Module Audio」は 強制指定 (Module Audio 未装着で「Module Audio」を選ぶと内蔵に フォールバックします)。保存後の再起動で切替わります。

Wi-Fi RTP 音声受信

Wi-Fi 経由の RTP 音声受信(L16/μ-law=udp/6970, AAC=udp/6972)。会話機能オン時は自動的に無効です

マイク リップシンク 調整 (会話・jtts どちらもオフのとき有効)

マイク入力の音量に合わせて口を開閉します。CoreS3 はマイク感度が低めなので入力ゲインを上げると見栄えします。スライダーを動かすと 即時反映 + 自動保存します。

周辺の雑音レベルを自動追跡して感度を調整します。OFF にすると固定閾値で判定。保存後の再起動で反映されます。

jtts ボイス

空欄や変更なしで「保存」を押すと現在の設定は維持されます。

やわらかめの目安: 開大比 0.65–0.75 / tilt 6–12 dB / 帯域幅 1.2–1.5 (+息成分 0.1–0.25)。いずれも v2 方式で有効 (帯域幅はクラシックでも有効)。

LT タイムキーパー

本体の「LT」タブで開始/停止。ここでは既定時間と通知ワードを設定します（保存しなくても送信で即反映、保存で再起動後も保持）。読みはひらがな（jtts は漢字を読めません）

会話プロバイダ・API キー

どの会話バックエンドを使うかを切り替えます。キー / 設定は下記すべてに保存できます。

OpenAI / Gemini の会話に使う指示文。XiaoZhi はサーバー側の人格を使うため無視されます。保存後の再起動で反映されます。

会話 API への WebSocket 接続に付与する追加ヘッダ。1 行 1 つ「ヘッダ名: 値」。Cloudflare Access のサービストークン等に使用。全プロバイダ共通。保存後の再起動で反映されます。

タッチ操作

応答再生中にアバター画面 (頭) をタップで AI 応答を中断します。タッチ センサが敏感で誤動作しやすいので、初期値はオフです。なでなで (前→中→後の連続タッチ) は本トグルとは独立で常に有効です。保存後の再起動で反映されます。

デバイス機能の有効/無効

アバター画面の左上に残量ゲージを表示します（保存後の再起動で反映）

起動直後の C5–E5–G5（スピーカー動作確認音）を鳴らすかどうか。OFF にすると無音起動（保存後の再起動で反映）

無効にするとサーボ電源を入れず、サーボタスクも起動しません（保存後の再起動で反映）。操作画面の脱力/復帰トグルとは別物で、こちらは NVS に永続化されます。

アバターの口が開くと LED が反応します (会話再生・jtts 発話・マイク リップシンク いずれの口元アニメーションでも有効)。OFF のときは設定した明るさで固定。保存後の再起動で反映されます。

ON のときの制御方式を選びます。「輝度」は口が開くほど LED 全体が明るくなる 従来動作。「レベル メーター」はネコミミの三角形 (LED1 = 底辺、LED5 = 頂点) を 5 段階のバー グラフとして点灯させます。保存後の再起動で反映されます。

カメラ (テスト撮影)

本体前面の GC0308 カメラで 1 枚撮影して表示します (640×480 カラー)。 露出安定化のため撮影に 1 秒ほどかかります。QR スキャン実行中は使えません。 「RAW 撮影」は ISP (AWB・色変換) をバイパスした Bayer モザイク (RGGB) を取得し、 「カラーバー」はセンサー内蔵テストパターンで転送経路を検証します。

センサー レジスタ (色調整用)

0x プレフィクス可。書込はセンサー再起動まで有効 (NVS 保存なし)。

サーボ原点・可動範囲 (Takao base 等の自作版向け)

サーボの取り付け向きや可動範囲はベース基板で異なります。原点(0..1023)と可動範囲(度)を 設定して「保存して再起動」で反映してください。既定は M5 base の値です。

範囲設定モードを ON にするとサーボの力が抜け、手で動かした位置を取り込めます。 中央位置で「Yaw 原点取込」、左端で「Yaw 最小取込」、右端で「Yaw 最大取込」を順にタップしてください。 取り込んだ値は上の入力欄に反映されます。最後に「保存 & 再起動」で確定します。

ファームウェア更新 (OTA)

ローカルの firmware.bin を選んで「書き込み」を押すと、新ファームを 別パーティションに書き込んで再起動します。Wi-Fi 経由は BLE より大幅に高速です。

最新の release をローカル ファイルで書き込む手順 (iPhone / AP モードでも可)
  1. Safari で ciniml.github.io/stackchan-idf を開く。
    (AP モードで Wi-Fi がインターネットに繋がっていなくても、iOS は セルラーへ自動 fallback します。繋がらない場合は一度 Wi-Fi を切ってからやり直してください。)
  2. 該当ボードの firmware-vX.Y.Z-<board>.zip をダウンロード
  3. 「ファイル」App で ZIP をタップして展開 → 中の stackchan_idf.bin が OTA ペイロード本体
  4. (AP モードの場合) device の AP に Wi-Fi 接続し、このページを再度開く
  5. 上の「ファイル選択」で stackchan_idf.bin を選び「書き込み」

※ firmware-vX.Y.Z-<board>.bin ではなく stackchan_idf.bin の方を選んでください。前者はブートローダ+ パーティション+アプリの結合イメージ (USB 初回書き込み用) で OTA には使えません。

⚠ Android で「ファイル選択」が反応しない場合:

  1. この画面が「ネットワークにログイン」ミニブラウザ (CaptivePortalLogin) で開いている場合、ファイル選択は動作しません。
  2. 右上の ⋮ メニュー →「このネットワークをこのまま使用」 を選択してください (Stackchan AP への接続が維持されます)。
  3. Chrome を開いて http://192.168.4.1/ にアクセスし、そちらで ファイル選択・OTA を実行してください。
※ Android が「インターネット接続がありません」と表示しても、 接続は維持したままにしてください。元の Wi-Fi に自動で戻って しまう場合は、Wi-Fi 設定で Stackchan AP を選び直してください。

release から直接取得 (device side fetch)

device が STA 経由で GitHub Pages から自分宛のビルドを取得し OTA 適用します。AP モード中でも STA がアップであれば動作 (APSTA)。STA が ダウンの場合はエラーになるので、上の「ローカル ファイル」を使ってください。

ドロップダウンで選ぶと右のテキスト欄に反映されます。手入力も可 (未来の tag / 特殊 build)。ボード判定 (cores3 / atoms3r / atoms3 / stopwatch) は device 側で自動。進捗は下の status バーに反映されます。

⚠ AP モード (STA 未接続) 中はリリース取得できません。上の「ローカル ファイル アップロード」を使用してください。
